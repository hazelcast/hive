TextField is the canonical control for short, single-line text input. Use it for
names, identifiers, search terms, URLs, emails and any other free-form value that
fits on one line. For multi-line content prefer `TextArea`; for numeric input
prefer `NumberField`; for secrets prefer `PasswordField`.

        <p>TextField renders a real <code>{'<input>'}</code> element. Most accessibility comes for free — the notes below cover edge cases.</p>

        <h2>Accessible name</h2>

        <p>The visible label is the accessible name (wired up via <code>htmlFor</code>). Never omit the label. If a design calls for a label-less field (rare), associate the input with an external label via <code>aria-labelledby</code>.</p>

        <h2>Placeholders are not labels</h2>

        <p>Placeholder text disappears once the user types and has poor contrast by design. Don&rsquo;t use it as the only descriptor of a field — screen readers may skip it and users with cognitive differences lose context.</p>

        <h2>Input type</h2>

        <p>Pick the <code>type</code> that matches the data — <code>email</code>, <code>url</code>, <code>tel</code>, <code>search</code>. Mobile keyboards adapt to the type, and browsers can validate basic patterns automatically.</p>

        <h2>Required fields</h2>

        <p>Mark required fields with <code>required</code>. The label renders an asterisk and the input gets <code>aria-required</code>. Avoid marking optional fields — clutter scales poorly.</p>

        <h2>Errors</h2>

        <p>When using <code>error</code>, the message is associated with the input via <code>aria-errormessage</code> and the input gets <code>aria-invalid="true"</code>. Screen readers announce the error on focus. Keep error text actionable.</p>

        <h2>Disabled fields</h2>

        <p>A disabled field is not focusable and is invisible to screen readers in forms mode. Only disable when the value is structurally locked. If the user needs to know <em>why</em>, surface that explanation in surrounding text.</p>

        <h2>Keyboard</h2>

        <ul>
          <li><code>Tab</code> moves focus into the field.</li>
          <li>A visible focus ring is shown on <code>:focus-visible</code>. <strong>Don&rsquo;t override it.</strong></li>
          <li>The clear button (when <code>clearable</code>) is reachable with <code>Tab</code> and activatable with <code>Enter</code> or <code>Space</code>.</li>
        </ul>

        <h2>Color contrast</h2>

        <p>The default border, label, placeholder and error colors all meet WCAG AA against the input background. Don&rsquo;t override them with custom CSS for decorative reasons.</p>

        <h2>Testing checklist</h2>

        <ul>

          <li>The <code>type</code> attribute matches the data.</li>

          <li>Error messages are actionable and announced on focus.</li>
